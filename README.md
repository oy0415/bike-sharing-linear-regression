# Bike Sharing データの単回帰分析

## 概要
このリポジトリは、Bike Sharing データを用いて、**気温 `temp` が利用者数 `cnt` に与える影響**を回帰分析で調べたノートブックをまとめたものです。  
まずは `cnt ~ temp` の単回帰モデルを作成し、その後 **2次項 `temp^2`** を加えたモデルと比較して、どちらがより適切かを検証しています。

分析では、データ確認、基本集計、可視化、単回帰、残差診断、モデル比較までを一通り行っています。

## 使用データ
- `day.csv`
  - Bike Sharing Dataset の日別集計データ
- 目的変数
  - `cnt` : 利用者数
- 主な説明変数
  - `temp` : 気温
  - `hum` : 湿度
  - `windspeed` : 風速

今回の中心テーマは、**気温と利用者数の関係**です。

## 分析内容
ノートブックでは、主に次の流れで分析を進めています。

1. データの読み込み
2. 欠損値・型の確認
3. 基本集計と可視化
   - カテゴリー変数ごとの平均確認
   - 箱ひげ図
   - 散布図
   - ヒストグラム
   - 相関行列とヒートマップ
4. 単回帰モデル `cnt ~ temp` の構築
5. 残差診断
   - 残差 vs 予測値
   - Q-Q plot
   - Scale-Location Plot
6. 2次項を含むモデル `cnt ~ temp + temp^2` との比較
7. AIC を用いたモデル評価
8. 結論の整理

## 主な結果
### 1. 単回帰モデル `cnt ~ temp`
- `temp` の係数は **正**
- p値は非常に小さく、**統計的に有意**
- 調整済み決定係数は **0.394**

このことから、**気温が高いほど利用者数が増える傾向**が確認されました。  
ただし、残差プロットでは LOWESS 線がやや山なりになっており、**単純な一次式だけでは関係を十分に捉えきれていない可能性**が示されました。

### 2. 2次項を加えたモデル `cnt ~ temp + temp^2`
- `temp` の係数: **21406.931**
- `temp^2` の係数: **-15055.025**
- 調整済み決定係数: **0.453**

`temp^2` の係数が負であることから、利用者数は気温とともに増えるものの、**増え方は一定ではなく、次第に緩やかになる可能性**が示唆されます。

### 3. AIC による比較
- Nullモデル: **13141.373**
- `cnt ~ temp`: **12777.536**
- `cnt ~ temp + temp^2`: **12704.116**

AIC は小さいほど良いため、今回比較した3つのモデルの中では  
**`cnt ~ temp + temp^2` が最も適切**という結果になりました。

### 4. 診断結果のまとめ
- 残差プロットでは、2次項を入れたモデルのほうが一次モデルより改善
- Q-Q plot でも、2次項ありモデルのほうが正規直線付近にやや集まりやすい
- 一方で、Scale-Location Plot では**等分散性の明確な改善までは確認できない**

## 結論
本分析では、**気温のみの単回帰モデルよりも、2次項を含むモデル `cnt ~ temp + temp^2` のほうが妥当**と判断しました。  

特に、
- AIC がより小さい
- 残差の曲線的な傾向が弱まる
- Q-Q plot でもわずかに改善がみられる

という点を総合して、**最終的に2次モデルを採用**しています。
